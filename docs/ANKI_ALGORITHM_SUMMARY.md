# Anki 间隔重复算法学习总结

> 本文档基于 Anki 官方手册整理，为本项目开发间隔重复复习功能提供参考。

---

## 一、核心概念

### 1.1 基础概念对照

| Anki 概念 | 对应本项目 | 说明 |
|-----------|-----------|------|
| **牌组 (Deck)** | **分组 (Group)** | 测验的组织单元，对应用户的分组功能 |
| **卡片 (Card)** | **测验 (Quiz)** | 最小学习单元，即一道测验题 |
| **笔记 (Note)** | 可忽略 | Anki 中多个卡片的来源，本项目简化设计 |
| **字段 (Field)** | 测验内容 | 问题、答案、提示等信息片段 |

### 1.2 卡片/测验状态

| 状态 | 定义 | 转换条件 |
|------|------|----------|
| **新卡片** | 从未学习过的测验 | 首次打开进入「学习中」 |
| **学习中** | 正在学习阶段的测验 | 完成学习阶段后进入「复习中」 |
| **待复习** | 已毕业，等待下次复习 | 到达复习时间进入「今日复习」 |
| **今日复习** | 今天需要复习的测验 | 复习后重新安排间隔 |
| **重学中** | 复习时失败，重新学习 | 完成重学后回到「待复习」 |
| **搁置** | 临时推迟到第二天 | 自动/手动搁置 |
| **暂停** | 长期暂停，不参与复习 | 手动暂停/解除 |

## 二、SM-2 间隔重复算法详解

### 2.1 学习阶段 (Learning Phase)

**目的**：新测验需要通过多次短间隔重复才能进入长期复习。

**初学间隔配置**（示例）：`1分钟 10分钟 1天`

| 操作 | 结果 |
|------|------|
| 重来 (1) | 返回第一阶段，1分钟后再次显示 |
| 良好 (3) | 进入下一阶段，按间隔延迟 |
| 简单 (4) | **直接毕业**，进入复习阶段 |

**毕业条件**：
- 完成所有初学阶段（按「良好」多次）
- 或直接按「简单」跳过

### 2.2 复习阶段 (Review Phase)

#### 核心公式

```
新间隔 = 旧间隔 × 简易度系数 × 评级乘数
```

| 评级 | 含义 | 乘数 | 建议使用频率 |
|------|------|------|-------------|
| **重来 (1)** | 完全忘记 | 重置为 1 天 | 5-20% |
| **困难 (2)** | 勉强想起 | 1.2-1.5 倍 | 偶尔使用 |
| **良好 (3)** | 正常回忆 | 正常乘数 | **80-95%** |
| **简单 (4)** | 毫不费力 | 1.3-1.5 倍 | 偶尔使用 |

#### 简易度系数 (Ease Factor)

- 默认：`2500`（即 2.5 倍）
- 重来会降低简易度（最低 1300）
- 简单会提高简易度（最高 5000）

**简易度影响**：
- 简易度 2500 → 按「良好」间隔 × 2.5
- 简易度 1300 → 按「良好」间隔 × 1.3（难卡片复习更频繁）

### 2.3 遗忘处理 (Relearning)

当复习卡片选择「重来」：

1. 卡片进入「重学中」状态
2. 使用「重学间隔」（如：`1分钟 10分钟`）
3. 完成重学后：
   - 间隔 = max(最小间隔, 原间隔 × 重来乘数)
   - 通常最小间隔为 1 天

## 三、关键算法参数

### 3.1 每日上限设置

| 参数 | 作用 | 建议值 |
|------|------|--------|
| 每日新卡片上限 | 控制新内容摄入 | 10-20 张 |
| 每日复习上限 | 平滑复习高峰 | 100-200 张 |

**注意**：学习 20 张新卡片 ≈ 增加 200 张每日复习量

### 3.2 间隔配置参考

```
初学间隔：1m 10m 1d      (新卡片学习阶段)
毕业间隔：1d             (毕业后首次复习)
简单间隔：4d              (按简单后的间隔)
重学间隔：1m 10m          (遗忘后的重新学习)
最小间隔：1d              (重学完成后最少等待)
```

### 3.3 高级参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 最大间隔 | 最长复习间隔 | 36500 天 |
| 初始简易度 | 新卡片起始难度 | 2500 |
| 简单复习乘数 | 按简单后的间隔增长 | 1.35 |
| 困难复习乘数 | 按困难后的间隔增长 | 1.2 |
| 重来复习乘数 | 按重来后的间隔增长 | 1.0 |
| 全局间隔乘数 | 整体调整所有间隔 | 1.0 |

## 四、关联卡片处理 (Sibling Cards)

### 4.1 问题场景
同一测验可能生成多个卡片（如：正反面、多个填空），如果短时间内连续出现，会相互提示答案。

### 4.2 搁置 (Burying) 机制

当学习一张卡片时，自动搁置其关联卡片：

| 类型 | 搁置行为 |
|------|----------|
| 新卡片关联 | 搁置其他新关联卡片到第二天 |
| 复习卡片关联 | 搁置其他复习关联卡片到第二天 |
| 跨日学习关联 | 可配置是否搁置 |

**注意**：
- 学习中的卡片不会被搁置（时间紧迫）
- 暂停和搁置互斥

## 五、筛选牌组 (Filtered Decks)

### 5.1 用途

- 考前突击复习
- 处理积压卡片
- 针对特定标签学习
- 提前复习

### 5.2 常用筛选条件

```
复习忘记的卡片：" rated:1:7 " (7天内按过重来)
提前复习：" is:due prop:due<7 " (7天内到期)
处理积压：" is:due prop:due>-7 " (刚逾期) / " prop:due<=-7 " (久逾期)
特定标签：" tag:重要 "
```

### 5.3 重新排程选项

- **启用**：在筛选牌组中的表现会影响原牌组计划
- **禁用**：纯预览模式，不改变任何状态

## 六、统计与追踪

### 6.1 卡片信息字段

| 字段 | 说明 |
|------|------|
| 位置 | 新卡片出现顺序 |
| 间隔 | 两次复习间的天数 |
| 简易度 | 间隔增长系数 (如 2500 = 2.5倍) |
| 复习次数 | 累计复习次数 |
| 遗忘次数 | 累计重来次数 |

### 6.2 重要统计图表

| 图表 | 用途 |
|------|------|
| 预测 | 未来到期复习量 |
| 日程 | 历史复习活动 |
| 复习次数 | 按熟练度分类的复习统计 |
| 复习间隔分布 | 卡片间隔分布情况 |
| 回答按钮 | 各评级按钮使用频率 |

### 6.3 关键指标

- **每日工作量**：平均每天需要复习的卡片数
- **预估习得总数**：当前可能记住的卡片总数
- **正确率**：未按重来的复习占比

## 七、FSRS 算法简介

**自由间隔重复排程算法 (Free Spaced Repetition Scheduler)** 是 SM-2 的替代方案。

### 7.1 核心改进

| 特性 | SM-2 | FSRS |
|------|------|------|
| 遗忘曲线 | 固定指数衰减 | 个性化模型 |
| 难度评估 | 简易度系数 | 难度 + 稳定性参数 |
| 记忆模型 | 简单间隔乘数 | 可提取性计算 |

### 7.2 FSRS 特有概念

- **稳定性**：记忆从 100% 降至 90% 所需时间
- **可提取性**：当前回忆正确的概率
- **期望保留率**：用户设定的目标记忆保留率

## 八、本项目实现建议

### 8.1 数据模型设计

```sql
quiz_review_status 表：
- quiz_id (FK)
- user_id (FK)
- status: enum(NEW, LEARNING, REVIEW, RELEARNING, SUSPENDED, BURIED)
- interval_days: int          -- 当前间隔（天）
- ease_factor: int            -- 简易度系数（默认 2500，即 2.5）
- next_review_date: date      -- 下次复习日期
- last_review_date: date      -- 上次复习日期
- review_count: int           -- 累计复习次数
- lapse_count: int            -- 遗忘次数
- learning_step: int          -- 当前学习阶段（0=未开始）
```

### 8.2 API 设计

| 端点 | 功能 |
|------|------|
| `GET /api/review/today` | 获取今日复习列表 |
| `POST /api/review/{id}/rate` | 提交评级 (1-4) |
| `POST /api/review/{id}/suspend` | 暂停/恢复测验 |
| `POST /api/review/{id}/bury` | 搁置测验到明天 |
| `GET /api/review/stats` | 学习统计数据 |
| `GET /api/review/forecast` | 未来复习预测 |

### 8.3 前端交互要点

1. **学习界面**：显示问题 → 显示答案 → 选择评级
2. **评级按钮**：显示预计下次复习时间
3. **每日提醒**：首页显示今日复习数量弹窗
4. **分组角标**：侧边栏显示各分组待复习数量

### 8.4 简化建议

对于 MVP 版本，可简化：
- 暂不支持 FSRS，先用 SM-2
- 简化为 3 个评级：重来/良好/简单
- 暂不支持复杂筛选牌组功能
- 优先实现核心学习流程
