# 敲脑壳 MindPop 极极简风设计原型 (Minimalist Zen)

## 1. 设计理念
- **空**: 移除所有非必要的装饰、阴影和渐变。
- **息**: 增加留白，让视觉有呼吸感，专注于文字本身。
- **恒**: 使用经典系统字体栈，保证在任何设备上的一致性。
- **信**: 极低饱和度的色彩体系，仅在关键交互（正确/错误）处使用色彩。

## 2. 全局规范 (Design Tokens)
- **背景**: `#FFFFFF` (纯白) 或 `#FAFAFA` (极浅灰)
- **文字**: 主色 `#171717` (近黑)，次要 `#737373` (中灰)
- **品牌色**: `#000000` (纯黑) - 用于按钮和强调
- **反馈色**: 
  - 正确: `#16A34A` (深绿)
  - 错误/放弃: `#DC2626` (深红)
  - 待复习: `#D97706` (琥珀色)
- **边框**: `1px solid #E5E5E5`
- **圆角**: `0px` (硬核极简) 或 `4px` (微圆角)

## 3. 核心页面原型描述

### 首页 (Home.html)
- **顶部**: 仅左侧一个文字 Logo "敲脑壳 MindPop"，右侧 "SIGN OUT"。
- **主体**: 居中对齐。
  - 大标题：`敲脑壳 MindPop` (加粗，大字号)
  - 副标题：`专注于知识内化的打字训练`
  - 核心操作：两个等宽的线性按钮 `[ START ]` `[ MANAGE ]`
- **底部**: 极细的分割线，下方是简单的文字链接。

### 测验列表 (Quizzes.html)
- **布局**: 单列居中窄版布局。
- **筛选**: 
  - **分类与分组并列**: 简单的文字链接，无边框，仅下划线。
  - 第一行：类型（全部/打字/填空）。
  - 第二行：分组（默认/编程基础/地理常识等）。
- **列表项**: 
  - 移除卡片感，改为列表感。
  - 每一项：标题 (左) | 数量 (右)
  - 悬停：背景变为 `#F5F5F5`，出现箭头。

### 答题界面 (Index.html)
- **顶部**: 细线进度条（0-100%）。
- **中间**: 
  - **测验标题**: 巨大细体文字 (如 `3rem`)，作为核心视觉焦点。
  - **计时器**: 位于标题下方，较小字号但加粗 (如 `1.5rem`)，灰色数字。
- **输入框**: 
  - 位于屏幕正中。
  - 仅一条下划线，无边框。
  - 字体加大，输入文字居中。
- **答案区**: 
  - 默认显示为灰色的占位点 `•` 或下划线。
  - 答对后文字直接替换占位符。

### 首页 (Home.html)
- **复习看板**: 置顶展示今日复习统计。
- **状态统计**: 完整保留原版统计维度：`新测验`、`待复习`、`学习中`、`已暂停`。
- **最近活动**: 列表展示最近练习记录（包含准确率、用时、时间戳）。
- **操作**: 居中的 `[ 开始复习流 ]` 线性按钮。

### 测验列表 (Quizzes.html)
- **模式切换**: 保留“单个测验”与“分组测验”的切换入口。
- **筛选**: 
  - 第一行：类型（全部/打字/填空）。
  - 第二行：分组（默认/编程基础等）。
- **列表项**: 包含标题、数量、时间限制、以及历史最佳准确率。

### 答题界面 (Index.html)
- **沉浸感**: 顶部 1px 进度条。
- **视觉重心**: 巨大细体测验标题 (`3rem`)，下方为灰色加粗计时器 (`1.5rem`)。
- **模式保留**:
  - **普通模式**: 核心单下划线输入，占位符 `•`。
  - **复习模式**: 底部浮现评级面板（重来/困难/良好/简单），包含间隔时间预览。
  - **题型支持**: 完整支持“打字题”与“填空题”的视觉呈现。

### 结算界面 (Results) - 内嵌式布局
- **核心原则**: 保留题目区域（标题、答案网格/填空原文），在下方内嵌结算统计。
- **布局**:
  - 题目区域保持可见，答案项标记为红色边框（未答出）或绿色（已答出）。
  - 结算区域位于题目下方，顶部 1px 分隔线 (`border-top`) 区分。
  - 复习评级面板固定在结算区域下方（如为复习模式）。
- **标题**: `测验完成`，细字重 (font-weight: 200)，字间距 0.3em，居中。
- **统计卡片**:
  - 三列等宽卡片横向排列，间距 1.5rem。
  - 卡片样式: 1px 边框，内部大字号数值 (2rem, font-weight: 200) + 小字标签 (0.65rem, 灰色)。
  - 显示内容: 准确率、答对数 (如 0/4)、总用时。
- **分隔线**: 1px 灰色水平线 (`results-divider`)，上下留白 2rem。
- **未答出区域**:
  - 标题: `未答出项目`，小字灰色居中，字间距 0.15em。
  - 列表项: 红色边框 (`var(--error)`)，flex 布局，答案内容在左，注释在右（灰色斜体）。
  - 间距: 项与项之间 0.5rem。
- **全部答对状态**:
  - 绿色边框容器，内含 ✓ 图标 + `全部答对` 文字。
  - 居中展示，替代未答出区域。
- **操作按钮**:
  - 双按钮布局: 主按钮（黑底白字 `.btn-primary`）+ 次按钮（灰色边框 `.btn-secondary`）。
  - 按钮间距 1rem，最小宽度 120px。
  - 功能: 重新开始、返回列表。

### 填空题结算界面 (Fill-Blank Results)
与打字题结算保持统一风格，增加填空题特有展示：
- **统计卡片**: 准确率、填空数 (如 3/4)、总用时。
- **填空回顾区域**:
  - 标题: `填空回顾`，小字灰色居中。
  - 原文展示: 带边框容器，内含完整原文。
  - 已填内容: 绿色实心下划线标记 `<span class="fb-filled">`。
  - 未填内容: 红色虚线方括号标记 `<span class="fb-missed-mark">[答案]`。
- **填空状态列表**:
  - 逐条列出每个填空的答题状态。
  - 正确项: 绿色边框，标注"正确"。
  - 错误项: 红色边框，标注"未答出"。
- **未填出详情**:
  - 标题: `未填出`。
  - 列表项: 左侧显示原文位置描述（如"常用于 ____ 开发"），右侧显示正确答案（红色加粗）。
- **全部填对状态**: 同打字题，绿色边框容器显示 ✓ 图标和"全部填对"。

### 创建与管理 (Create.html / Manage.html)
- **创建页**: 
  - **导入**: 包含“导入 JSON”快速入口。
  - **通用字段**: 标题 (必填)、描述、时间限制 (秒)、所属分组下拉选择。
  - **打字题**: 批量录入答案，支持 `#注释#` 格式，提供"插入注释"快捷按钮。
  - **填空题**: 完整支持原版录入逻辑，提供"挖空选中文字 ([])"和"插入注释 (#)"两个快捷按钮，并配备实时填空预览区域。
- **管理页**: 
  - **批量操作**: 保留“全选”、多选框、批量删除、批量移动分组。
  - **数据维护**: 包含分组列表管理与单体测验列表维护。
  - **导出**: 提供“导出全部备份”按钮。

### 创建页 (Create.html) & 管理页 (Manage.html)
- **导航独立**: “创建”与“管理”在导航栏并列。
- **创建页**: 纯净表单，支持标题、描述、类型切换及批量答案录入。
- **管理页**: 仅保留分组列表与测验列表的增删改查，移除数据库冗余信息。

---

## 4. 交互动效
- **淡入淡出**: 页面切换使用 200ms 的淡入淡出。
- **无位移**: 尽量避免复杂的位移和缩放动画，保持静谧感。
- **删除**: 删除了原有的“数据库管理”页面。
