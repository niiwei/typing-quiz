# 间隔重复复习系统 - 设计与方案文档

本文档整合Anki算法原理和产品功能方案，为复习系统的设计与开发提供完整参考。

---

## 一、核心概念

### 1.1 Anki概念映射

| Anki概念 | 本系统映射 | 说明 |
|---------|-----------|------|
| **牌组 (Deck)** | **分组 (Group)** | 测验的组织单元 |
| **卡片 (Card)** | **测验 (Quiz)** | 最小学习单元 |
| **学习阶段** | **首次学习** | 新测验需完成首次学习才能进入复习 |
| **复习阶段** | **间隔重复** | 根据算法安排复习时间 |
| **暂停/搁置** | **测验状态** | 暂停=不再安排；搁置=延迟复习 |

### 1.2 测验状态流转

| 状态 | 定义 | 转换条件 |
|------|------|----------|
| **新测验** | 从未学习过 | 首次打开进入「学习中」 |
| **学习中** | 正在首次学习阶段 | 完成后进入「待复习」 |
| **今日复习** | 算法安排今天复习 | 复习后重新安排间隔 |
| **待复习** | 已安排未来日期 | 到达时间进入「今日复习」 |
| **重学中** | 复习时失败 | 完成后回到「待复习」 |
| **搁置** | 临时推迟 | 可设1/3/7天后恢复 |
| **暂停** | 长期暂停 | 不纳入复习队列 |

## 二、SM-2 算法详解

### 2.1 核心公式

```
新间隔 = 旧间隔 × 简易度系数 × 评级乘数
```

### 2.2 评级体系

| 评级 | 含义 | 乘数 | 使用频率 |
|------|------|------|---------|
| **重来 (1)** | 完全忘记 | 重置为 1 天 | 5-20% |
| **困难 (2)** | 勉强想起 | 1.2-1.5 倍 | 偶尔 |
| **良好 (3)** | 正常回忆 | 正常乘数 | **80-95%** |
| **简单 (4)** | 毫不费力 | 1.3-1.5 倍 | 偶尔 |

### 2.3 简易度系数 (Ease Factor)

- **默认值**：2500（即 2.5 倍）
- **重来后**：降低简易度（最低 1300）
- **简单后**：提高简易度（最高 5000）
- **影响**：简易度越低，复习越频繁

### 2.4 遗忘处理 (Relearning)

当复习选择「重来」：
1. 进入「重学中」状态
2. 使用重学间隔（如：`1分钟 10分钟`）
3. 完成后间隔 = max(1天, 原间隔 × 重来乘数)

## 三、算法配置

### 3.1 间隔配置参考

```
初学间隔：1m 10m 1d      (新卡片学习阶段)
毕业间隔：1d              (毕业后首次复习)
简单间隔：4d              (按简单后的间隔)
重学间隔：1m 10m          (遗忘后重新学习)
最小间隔：1d              (重学完成后最少等待)
```

### 3.2 关键参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 初始简易度 | 新卡片起始难度 | 2500 |
| 最大间隔 | 最长复习间隔 | 36500 天 |
| 简单复习乘数 | 按简单后间隔增长 | 1.35 |
| 困难复习乘数 | 按困难后间隔增长 | 1.2 |
| 每日新卡片上限 | 控制新内容摄入 | 10-20 张 |
| 每日复习上限 | 平滑复习高峰 | 100-200 张 |

**注意**：学习 20 张新卡片 ≈ 增加 200 张每日复习量

## 四、产品方案

### 4.1 解决的痛点

| 痛点 | 解决方案 |
|------|----------|
| **学了就忘** | 间隔重复算法在遗忘临界点推送复习 |
| **不知复习什么** | 自动计算最佳复习时间 |
| **新内容被淹没** | 区分"学习"和"复习"阶段，优先新内容 |

### 4.2 用户旅程示例

**Day 1 - 导入新测验**
- 侧边栏看到分组有 [5] 个待学习
- 打开测验完成首次学习，选择"正常"评级
- 系统提示"3天后复习"

**Day 4 - 首次复习**
- 首页弹窗提示有测验待复习
- 练习后评级"困难"，间隔缩短
- 状态变为"待复习(2天后)"

**Week 2 - 批量复习**
- 首页显示"今日复习 12 个"
- 使用批量复习模式连续练习
- 查看统计图表了解记忆曲线

### 4.3 推送机制

- 每日首次登录显示"今日复习"弹窗
- 侧边栏显示各分组复习数量角标
- 分组页面顶部显示待复习数量

### 4.4 预计障碍与缓解

1. **算法黑盒感** → 显示上次练习日期和下次复习日期
2. **学习负担不均** → 提供"每日上限"设置，超限顺延
3. **误操作惩罚** → 增加"修改上次评级"功能允许纠错

## 五、技术实现

### 5.1 技术栈

| 层级 | 技术选型 | 理由 |
|------|----------|------|
| 算法计算 | Java 服务端 | SM-2 算法在后台计算 |
| 状态存储 | MySQL + quiz_review_status 表 | 与现有架构一致 |
| 推送触发 | Spring Scheduler (每日 00:01) | 生成推送记录 |
| 前端提醒 | JS 弹窗 + LocalStorage | 避免重复打扰 |
| 复习队列 | 内存缓存 (可选 Redis) | 性能优化 |

### 5.2 数据模型

```sql
quiz_review_status 表:
- quiz_id (FK)
- user_id (FK)
- status: enum(NEW, LEARNING, REVIEW, RELEARNING, SUSPENDED)
- interval_days: int          -- 当前间隔天数
- ease_factor: int            -- 简易度系数 (默认 2500)
- next_review_date: datetime   -- 下次复习日期
- last_review_date: datetime   -- 上次复习日期
- review_count: int           -- 累计复习次数
- lapse_count: int             -- 遗忘次数
- learning_step: int           -- 当前学习阶段
```

**索引设计**：`(user_id, next_review_date)` - 用于今日复习列表查询

### 5.3 核心流程

**每日任务 (Scheduler @ 00:01)**：
1. 对每个用户查询 next_review_date = 今天的测验
2. 更新状态为今日待复习
3. 生成分组汇总统计

**用户练习后处理**：
1. 若首次学习 → 设置初始间隔
2. 若复习阶段 → 计算新间隔 = 旧间隔 × ease_factor × rating_multiplier
3. 更新 next_review_date = today + interval
4. 调整 ease_factor (评级好则↑，差则↓)

### 5.4 API 端点

| 端点 | 说明 |
|------|------|
| `GET /api/review/today` | 获取今日复习列表 |
| `POST /api/review/{id}/rate` | 提交评级 (1-4) |
| `POST /api/review/{id}/suspend` | 暂停/恢复测验 |
| `POST /api/review/{id}/bury` | 搁置测验到明天 |
| `GET /api/review/stats` | 学习统计数据 |
| `GET /api/review/forecast` | 未来复习预测 |

## 六、MVP 简化建议

为快速上线，可简化：
- 暂不支持 FSRS 算法，先用 SM-2
- 简化为 3 个评级：重来/良好/简单
- 暂不支持复杂筛选牌组功能
- 优先实现核心学习流程
