# 开发流程规范

## 代码提交流程

1. **本地修改代码**
2. **本地测试验证**
3. **测试通过后提交并推送**
4. **部署到云服务器**

## 测试要求

- 功能测试通过后再提交
- 确保不影响现有功能
- 本地环境能复现的问题不在云服务器上调试

## Git 使用规范

```bash
# 提交前确认测试通过
git add -A
git commit -m "fix: xxx"
git push
```

## 环境配置

- 本地连接云服务器 MySQL：`MYSQL_HOST=47.102.147.127`
- 云服务器使用 `localhost`

## 账户隔离功能

- 测验：按 userId 过滤
- 分组：按 userId 过滤
- 前端需发送 Authorization header
